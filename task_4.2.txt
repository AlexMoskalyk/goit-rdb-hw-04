
Коли мы заменяем операторы INNER JOIN на LEFT JOIN или RIGHT JOIN, мы изменяем логику объединения таблиц.

INNER JOIN возвращает только те строки, которые имеют соответствующие значения в обеих таблицах. Если пара не найдена, то результаты из обеих таблиц для этой строки не будут включены в итоговый набор данных.

LEFT JOIN (или LEFT OUTER JOIN) возвращает все строки из "левой" таблицы (то есть, указанной первой в SQL запросе) и совпадающие строки из "правой" таблицы (второй в запросе). Если совпадение по второй таблице не найдено, результат все равно будет включать строки из первой таблицы, но с NULL в столбцах второй таблицы.

RIGHT JOIN (или RIGHT OUTER JOIN) работает как LEFT JOIN, но в обратном порядке: возвращаются все строки из "правой" таблицы и совпадающие из "левой". Если совпадение по "левой" таблице не найдено, в итоговый набор данных попадают строки из "правой" таблицы с NULL в столбцах "левой".

В результате использования LEFT JOIN или RIGHT JOIN количество строк в результирующем наборе может увеличиться по сравнению с INNER JOIN. Это происходит потому, что теперь в результат включаются строки, для которых нет совпадений в одной из таблиц. Это полезно, когда нам нужно сохранить все данные из одной таблицы, даже если соответствующие данные в другой таблице отсутствуют.
